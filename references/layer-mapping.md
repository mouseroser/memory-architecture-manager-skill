---

### 数据流向映射

#### 写入流向

```
用户对话
    ↓
[Layer 2] memory-lancedb-pro 自动捕获
    ↓
[Layer 1] 人工提炼到 MEMORY.md
    ↓
[Layer 3] Cron 同步到 NotebookLM
```

#### 检索流向

```
用户查询
    ↓
[Layer 2] memory_recall (90% 场景)
    ├─ 命中 → 返回结果
    └─ 未命中 ↓
[Layer 3] NotebookLM 查询（深度理解）
    ├─ 命中 → 返回结果
    └─ 未命中 ↓
[Layer 1] 直接读取文件（原始记录）
```

### 迁移映射

| 迁移场景 | 方案 |
|----------|------|
| **Layer 1 → Layer 2** | 手动导入：`memory-pro import` |
| **Layer 2 → Layer 1** | Markdown mirror 已自动同步 |
| **Layer 1 → Layer 3** | Cron 任务自动同步 |
| **Layer 3 → Layer 1** | NotebookLM 导出 → 本地文件 |
| **完整备份** | Layer 1 (Git) + Layer 2 (Markdown mirror) + Layer 3 (云端) |
| **完整恢复** | Layer 1 (Git restore) → Layer 2 (重建索引) → Layer 3 (重新同步) |

## 决策树

### 何时使用哪一层？

```
需要快速检索？
├─ 是 → Layer 2 (memory_recall)
└─ 否 ↓

需要深度理解？
├─ 是 → Layer 3 (NotebookLM)
└─ 否 ↓

需要人工查看？
├─ 是 → Layer 1 (直接读文件)
└─ 否 ↓

需要跨 agent 共享？
├─ 是 → Layer 1 (shared-context/)
└─ 否 ↓

需要长期归档？
└─ 是 → Layer 3 (NotebookLM)
```

### 何时维护哪一层？

```
每天？
└─ Layer 2 健康检查

每周？
├─ Layer 1 压缩检查
├─ Layer 1 MEMORY.md 维护
└─ Layer 3 周度同步

每月？
├─ Layer 3 月度归档
└─ Layer 3 深度洞察
```
